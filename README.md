# نظام متابعة - FollowUp System

مشروع تخرج لإدارة عمال البلدية.

## ما هو المشروع؟

نظام لإدارة عمال النظافة والصيانة في بلدية البيرة. يسمح للعمال بتسجيل الحضور والإبلاغ عن المشاكل، ويسمح للمشرفين بمتابعة العمال وتوزيع المهام.

## مكونات المشروع

```
FollowUp-Repo/
├── backend/          # ASP.NET Core 9 API
├── mobile/           # Flutter تطبيق الموبايل
├── web/              # React لوحة تحكم المشرفين
└── AzizaWEB/         # HTML/JS صفحات بسيطة (بديل)
```

## معمارية النظام - Platform Roles

يتبع النظام معمارية متعددة المنصات حسب دور المستخدم:

### **تطبيق الموبايل (Mobile)** - خاص بالعمال فقط
- **الأدوار المدعومة:** عامل (Worker) فقط
- **الوظائف:**
  - تسجيل الحضور والانصراف بالـ GPS
  - استقبال وإتمام المهام
  - رفع البلاغات مع الصور
  - تقديم الطعون على القرارات
  - عرض الإشعارات

### **لوحة التحكم الويب (Web)** - لإدارة النظام والإشراف
- **الأدوار المدعومة:** مدير (Admin) + مشرف (Supervisor)
- **وظائف المدير (Admin):**
  - إدارة الحسابات (إنشاء مستخدمين، حذف، تعديل)
  - إدارة الأقسام والفرق
  - إدارة المناطق الجغرافية
  - إدارة المشرفين
  - مراجعة الطعون
  - التقارير وسجلات النظام
- **وظائف المشرف (Supervisor):**
  - عرض العمال التابعين له
  - إنشاء وتوزيع المهام (فردية أو جماعية)
  - متابعة موقع العمال (GPS Tracking)
  - مراجعة أداء العمال
  - عرض الفرق (قراءة فقط - للمشرف حق الاطلاع فقط)

### **السيرفر الخلفي (Backend)** - API موحد
- يخدم كل من الموبايل والويب
- التحقق من الصلاحيات على مستوى الـ endpoints
- بعض الـ endpoints خاصة بالمشرفين/المدراء فقط

**ملاحظة:** هذا التصميم متعمد - تطبيق الموبايل خاص بالعمال الميدانيين، بينما لوحة التحكم الويب للإدارة والإشراف من المكتب.

## تصميم إدارة الحضور (Attendance Management Design)

يتبع نظام الحضور في FollowUp معمارية **Mobile-First**:

### ✅ تنفيذ الحضور: الموبايل فقط (Execution)
- **تسجيل الحضور والانصراف** يتم حصرياً عبر تطبيق الموبايل
- **يتطلب تسجيل الحضور:**
  - إحداثيات GPS دقيقة
  - التحقق من المنطقة الجغرافية (Geofencing)
  - ربط الجهاز (Device Binding)
  - التحقق من الموقع الفعلي للعامل

**هذه المتطلبات غير متوفرة بشكل موثوق في متصفحات الويب.**

### ✅ مراجعة الحضور: لوحة التحكم الويب (Reporting)
- **المشرفون والمدراء** يستخدمون لوحة التحكم الويب لـ:
  - عرض تقارير الحضور (PDF / Excel)
  - تحليل إحصائيات الحضور
  - تدقيق السجلات التاريخية
  - تصدير بيانات رسمية للمحاسبة

### 🎯 فوائد هذا التصميم:
1. ✅ **دقة التحقق الميداني** - منع التلاعب بالموقع
2. ✅ **فصل واضح** بين العمليات الميدانية والتقارير الإدارية
3. ✅ **أمان أعلى** - لا يمكن للمشرفين تزوير حضور عامل بدون GPS
4. ✅ **سهولة الاستخدام** - العمال لا يحتاجون للويب، والمشرفون لا يحتاجون GPS

**ملاحظة:** تم تصميم النظام بهذا الشكل عمداً. واجهة إدارة الحضور على الويب غير مطلوبة ولن يتم تطويرها.

## كيفية التشغيل

### 1. Backend (السيرفر)

```bash
cd backend/FollowUp.API

# تعيين connection string
dotnet user-secrets set "ConnectionStrings:DefaultConnection" "Server=localhost;Database=FollowUpDB;..."

# تعيين JWT secret
dotnet user-secrets set "JwtSettings:SecretKey" "your-secret-key-32-characters-min"

# تشغيل
dotnet run
```

السيرفر يعمل على: `http://localhost:5000`

### 2. Mobile (تطبيق الموبايل)

```bash
cd mobile

# تحميل المكتبات
flutter pub get

# تشغيل
flutter run
```

تأكد من تغيير `baseUrl` في `lib/core/config/api_config.dart` ليشير للسيرفر.

### 3. Web Dashboard (لوحة التحكم)

```bash
cd web

# تحميل المكتبات
npm install

# تشغيل
npm run dev
```

## قاعدة البيانات

المشروع يستخدم SQL Server مع Entity Framework. عند أول تشغيل، يتم إنشاء الجداول تلقائياً.

### استيراد مناطق العمل (GIS)

ضع ملف الـ shapefile في:
```
backend/FollowUp.API/GisData/your-shapefile.shp
```

المناطق تُستورد تلقائياً عند بدء التشغيل إذا كانت الجداول فارغة.

## المستخدمين الافتراضيين

بعد تشغيل السيرفر، استخدم Swagger أو endpoint الـ register لإنشاء أول مستخدم Admin.

```
POST /api/auth/register
{
  "username": "admin",
  "password": "Admin123!",
  "fullName": "مدير النظام",
  "role": "Admin"
}
```

## المتطلبات

- .NET 9 SDK
- SQL Server (LocalDB يكفي للتطوير)
- Flutter 3.x
- Node.js 18+

## ملاحظات للتطوير

- `DeveloperMode:DisableGeofencing = true` في appsettings.json يسمح بتسجيل الدخول من أي موقع (للاختبار)
- كل الـ API endpoints تتطلب JWT token ما عدا `/api/auth/login` و `/api/health/ping`
